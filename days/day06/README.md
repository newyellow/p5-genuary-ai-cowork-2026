# Day 06 - Lights On/Off

**Prompt:** Lights on/off - Make something that changes when you switch on or off the "digital" lights

**Concept:** Visibility & Privilege — 誰被看見，誰在黑暗中消失？

## 核心概念

### 光明與黑暗中的可見性

100 個人形在畫面中，但當燈光開關時，發生了不同的事：

**燈光開啟時：**
- 所有人都被照亮，充滿色彩
- 離光源近的人更明亮
- 20% 的「特權者」（金色星形頭）特別明亮
- 人們伸手向光，尋求照耀

**燈光關閉時：**
- 80% 的人幾乎消失在黑暗中（深色剪影）
- 20% 的「特權者」依然可見（微弱發光）
- 人們蜷縮身體，保護性姿態

### 社會可見性的隱喻

**誰總是被看見？**
- 金色星星（20%）= 社會中的特權階級
- 無論環境如何，他們總是可見、被關注
- 較大的體型，獨特的形狀（星形）

**誰在黑暗中消失？**
- 其他 80% = 邊緣群體、普通人
- 只有在「光明」條件下才被看見
- 環境惡化時，第一個被忽視的

### Meta 敘事：系統性不平等

**照明系統作為社會系統**
- 燈光不是中性的，它選擇性地照亮
- 距離光源的遠近 = 社會資本的多寡
- 系統決定誰被看見、誰的聲音被聽見

**可見性即存在**
- 在數位時代，不被看見 = 不存在
- 演算法的「燈光」選擇性地照亮某些內容
- Social media 的 visibility = 現代的光明/黑暗

## 技術實作

### 分層可見性系統

**人形分類**
- `alwaysVisible: true` (20%) - 特權者
  - 金色色相（hue 50）
  - 星形頭部
  - 較大尺寸（0.8-1.2x）

- `alwaysVisible: false` (80%) - 一般群眾
  - 藍/紫色相（hue 180-280）
  - 圓/方/三角頭
  - 較小尺寸（0.5-0.9x）

### 光照系統

**燈光開啟**
- 光源在上方中央，發出光暈和光線
- 亮度根據距離衰減：`distFactor = map(distance, 0, max, 1, 0.3)`
- 特權者不受距離影響，始終明亮

**燈光關閉**
- 一般人變成深色剪影（HSB: 0, 0, 6-10）
- 特權者保持微弱可見（HSB: hue, 30, 40）
- 描邊也變暗

### 動畫與姿態

**燈光開啟時**
- 人們伸手向光（手臂指向光源）
- 積極的姿態，尋求照耀
- 輕柔的呼吸動畫

**燈光關閉時**
- 手臂縮回，保護性姿態
- 身體微微蜷縮
- 在黑暗中等待

## 互動

- **點擊任意處** 或 **按空白鍵**：切換燈光
- **按 'S' 鍵**：儲存當前畫面

## 哲學反思

### 可見性政治

這件作品探討：

**1. 結構性特權**
- 為什麼有些人無論環境如何都被看見？
- 為什麼有些人只在特定條件下才可見？
- 這種差異是偶然的，還是系統性的？

**2. 環境與可見性**
- 當環境變差（燈光關閉），誰最先消失？
- 誰有資源在黑暗中依然發光？
- 可見性是權利還是特權？

**3. 數位可見性**
- 演算法決定誰被看見（推薦系統的「燈光」）
- 社交媒體的 verified badge = 始終可見的「星形」
- 影子禁令 (shadow ban) = 燈光關閉

**4. 努力與系統**
- 一般人伸手向光，努力被看見
- 但系統決定了誰能真正獲得照明
- 個人努力 vs. 結構性優勢

### 開關的權力

**誰控制開關？**
- 在這個作品中，觀者控制燈光
- 在現實中，誰控制可見性的開關？
- 媒體、演算法、政策制定者

**關燈的後果**
- 不是所有人受到相同影響
- 特權者依然發光
- 邊緣群體完全消失

> "In the darkness, some still shine while others vanish entirely.
> Visibility is not equally distributed."
